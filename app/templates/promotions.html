Create randomizer

Prize slices, routed journeys, segment and the visual bundle come from the captured template — only the dates & name change. Optionally override the weights and the routed journeys.

Type
Promo date(s)

Pick a date from the calendar and click Add — repeat for as many promos as you want. Each date becomes one randomizer, all created in one paste.

Days

Window length (applied to every date). Defaults per type (Scratch = 2).

Weights

Space-separated, in prize order. Must match the template's prize count.

Journeys

Space-separated routed journey ids, in prize order. Leave blank to keep the template's.

The script creates the randomizer draft then fills it. Set PREVIEW=true at the top to inspect the request bodies before sending.

How it works

A randomizer is a weighted set of prize slices; each slice routes a winning player to a journey. This keeps the captured prizes, segment and visual, and only re-dates + re-names the promo for a new run.

The two-step create flow (create draft → fill) is inferred from the captured bodies. If a call 404s, the draft is easy to delete — send me that request and I'll correct the endpoint.
